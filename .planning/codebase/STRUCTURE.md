## Directory Purposes

**`backend/`:**
- Purpose: FastAPI backend application
- Contains: Python source code, requirements, container config
- Key files: `main.py` (entry point), `converters/` (plugin system)
- Entry point: `python -m uvicorn main:app --host 0.0.0.0 --port 8070`

**`backend/converters/`:**
- Purpose: Pluggable format conversion modules
- Contains: One Python file per media category, plus registry/interface
- Key files: `__init__.py` (dispatcher), `base.py` (interface), category modules
- Pattern: Each module is a `BaseConverter` implementation with sync + async methods

**`frontend/`:**
- Purpose: User interface
- Contains: Single HTML file with embedded CSS and JavaScript
- Key files: `index.html` (everything)
- Pattern: Vanilla DOM manipulation, dark theme (Glacier + Dusk tokens)

**`.planning/codebase/`:**
- Purpose: GSD-generated documentation
- Contains: Architecture, structure, conventions, testing, concerns analysis
- Key files: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
- Generated by: `/gsd:map-codebase` command

**`.github/`:**
- Purpose: CI/CD pipelines
- Contains: GitHub Actions workflows
- Key files: `workflows/*.yml`

## Naming Conventions

**Files:**

- `.py` files: snake_case without hyphens
  - Modules named by category: `documents.py`, `images.py`, `audio.py`, `video.py`, `data.py`, `archives.py`
  - Main app file: `main.py`
  - Base interface: `base.py`
  - Registry: `__init__.py`

- `.html`, `.json`, `.yml` files: kebab-case or descriptive names
  - `index.html` (frontend SPA)
  - `docker-compose.yml` (Docker orchestration)
  - `nginx.conf` (web server config)
  - `requirements.txt` (Python dependencies)

**Directories:**

- lowercase, plural when containing multiple items: `converters/`, `backend/`, `frontend/`
- special directories: `.github/`, `.planning/` (dotfiles for tooling)

**Python Functions:**

- Private (module-level): `_snake_case_with_underscore_prefix`
  - Examples: `_store_upload()`, `_do_one_conversion()`, `_cleanup_loop()`, `_purge_expired()`

- Public (exported): `snake_case` without leading underscore
  - Examples: `detect_format()`, `convert_file()`, `get_available_formats()`

**Python Classes:**

- `PascalCase`: `BaseConverter`, `DocumentConverter`, `ImageConverter`, `AudioConverter`, `VideoConverter`, `DataConverter`, `ArchiveConverter`

**Python Variables:**

- Constants: `UPPER_SNAKE_CASE`
  - Examples: `UPLOADS`, `DOWNLOADS`, `TEMP_DIR`, `FILE_TTL`, `CLEANUP_INTERVAL`, `_VALID_QUALITY`

- Module-level dicts: `_UPPER_SNAKE_CASE` (leading underscore)
  - Examples: `_EXTENSIONS`, `_PYDUB_WRITE`, `_EXT_MAP`

- Local variables: `snake_case`
  - Examples: `file_id`, `target_format`, `input_path`, `output_path`

## Where to Add New Code

**New Converter Category (e.g., 3D models, eBooks):**

1. Create file: `backend/converters/newcategory.py`
2. Implement class extending `BaseConverter`:
   ```python
   from .base import BaseConverter

   class NewCategoryConverter(BaseConverter):
       @property
       def supported_formats(self) -> list[str]:
           return ["fmt1", "fmt2", ...]

       def format_extensions(self, fmt: str) -> list[str]:
           return {...}.get(fmt, [])

       def get_output_formats(self, input_format: str) -> list[str]:
           return [...]

       async def convert(self, input_path, input_format, output_format, output_path, **kwargs) -> None:
           loop = asyncio.get_running_loop()
           await loop.run_in_executor(None, self._convert_sync, ...)

       def _convert_sync(self, ...) -> None:
           # Sync implementation using third-party libs
   ```
3. Register in `backend/converters/__init__.py`:
   - Add import: `from .newcategory import NewCategoryConverter`
   - Add to registry dict: `"newcategory": NewCategoryConverter(),`
4. Extension map auto-populates at import time via loop (lines 33-36)

**New API Endpoint:**

1. Add route handler to `backend/main.py` after line 186 (endpoint section)
2. Use existing patterns:
   - Single file: `@app.post("/api/endpoint")` with `UploadFile = File(...)`
   - Form data: `@app.post("/api/endpoint")` with `Form(...)` parameters
   - Query params: `@app.get("/api/endpoint/{param}")`
3. Return JSON dict or `FileResponse` for downloads
4. Validate inputs and catch exceptions as HTTPException (status_code, detail)
5. Log at INFO level for significant operations
6. Use UUID for file IDs: `str(uuid.uuid4())`

**Frontend Component:**

1. Edit `frontend/index.html`
2. Add HTML structure in `<body>`
3. Add CSS to `<style>` block (use design tokens in `:root`)
4. Add JavaScript to `<script>` block:
   - Query selectors: `document.querySelector(selector)`
   - Event listeners: `element.addEventListener('event', handler)`
   - Fetch API: `fetch('/api/endpoint', {method: 'POST', ...})`
   - DOM manipulation: `element.textContent`, `element.classList`, `element.innerHTML`

**New Quality Preset (audio/video):**

1. Edit converter file (`backend/converters/audio.py` or `video.py`)
2. Add quality value to codec/bitrate map:
   - Audio: Update `_MP3_STANDARD_KBPS` or equivalent, `_QUALITY_PRESET` dict
   - Video: Update `_CRF` dict with new quality level
3. Update main.py `_VALID_QUALITY` set to include new value
4. Quality is passed via `quality` kwarg through `convert_file()` → converter

**Shared Utilities:**

- Currently: No shared utility module (all logic in converter classes or main.py)
- If adding: Create `backend/utils.py` and import in main.py or converters as needed
- Pattern: Functions in utils, no state, pure or side-effect-free where possible

## Special Directories

**`backend/__pycache__/`:**
- Purpose: Python bytecode cache
- Generated: Yes (automatic when Python imports)
- Committed: No (in .gitignore)

**`backend/.venv/`:**
- Purpose: Virtual environment (local dev only)
- Generated: Yes (created by `python -m venv .venv`)
- Committed: No (in .gitignore)
- Note: Not used in Docker; container uses system Python + pip install

**`TEMP_DIR` (runtime):**
- Purpose: Temporary file storage during uploads/conversions
- Created at: App startup via `tempfile.mkdtemp(prefix="fc_")`
- Location: System temp directory (typically `/tmp` on Unix)
- Cleaned up: On app shutdown via `shutil.rmtree(TEMP_DIR, ignore_errors=True)`
- Lifetime: Individual files expire after `FILE_TTL` (3600s) or after download
- Committed: N/A (runtime only)

**`.github/workflows/`:**
- Purpose: CI/CD automation
- Generated: No (human-written YAML files)
- Committed: Yes

**`.planning/codebase/`:**
- Purpose: GSD mapping output
- Generated: Yes (written by `/gsd:map-codebase`)
- Committed: Yes (planning documents)
- Contents: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md
