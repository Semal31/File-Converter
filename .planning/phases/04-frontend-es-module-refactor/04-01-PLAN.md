1. Create `frontend/css/` directory and `frontend/css/app.css`.
   - Copy the ENTIRE contents of the `<style>` block (lines ~11-499 of index.html) into `frontend/css/app.css`.
   - Preserve every rule exactly as-is — no modifications, no reordering, no minification. This is a pure extraction.
   - Include the design tokens (:root), all component styles, responsive media queries, animations/keyframes.

2. Create `frontend/js/` directory and `frontend/js/state.js`.
   - Extract all shared mutable state variables from the script block (lines ~787-801 of index.html):
     - `currentQuality` (default `'original'`)
     - `singleFileId` (default `null`)
     - `singleFormat` (default `null`)
     - `singleCategory` (default `null`)
     - `singleFmts` (default `[]`)
     - `bulkFiles` (default `[]`)
     - `history` (default `[]`) — keep name as `history` per research recommendation (module scope avoids window.history conflict)
   - Export all variables as `export let`.
   - Create and export mutation functions (since `export let` bindings are read-only from importers):
     - `setQuality(q)` — sets currentQuality, also updates quality button active states in DOM (query `#quality-opts .quality-opt` buttons, toggle `.active` class based on `data-q` matching `q`). This matches the existing `setQuality()` behavior.
     - `setSingleFile(id, fmt, cat, fmts)` — sets singleFileId, singleFormat, singleCategory, singleFmts
     - `resetSingleState()` — resets singleFileId/singleFormat/singleCategory to null, singleFmts to []
     - `resetBulkState()` — resets bulkFiles to []
     - `setBulkFormat(i, value)` — sets bulkFiles[i].target = value (the format selector handler for bulk rows)
     - `addBulkFile(fileObj)` — pushes to bulkFiles array
     - `getBulkFiles()` — returns bulkFiles reference (needed since importers cannot read `export let` arrays that get reassigned)
     - `addToHistory(entry)` — unshifts {entry + ts: Date.now()} into history, caps at 50 entries
     - `clearHistoryState()` — resets history to []
   - Use `.js` extension in all import paths (browser requirement).
   - Do NOT import from any other module — state.js is a leaf dependency.

IMPORTANT: Read the existing index.html carefully to capture the exact current behavior of setQuality() — it toggles `.active` class on quality buttons. This DOM manipulation stays in state.js for now because it is tightly coupled to the state change. (Alternative: move to ui.js and have main.js call both — use your judgment, but keep setQuality atomic.)

2. Create `frontend/js/ui.js`.
   - Extract all DOM helper and rendering functions from the existing script block.
   - Import from `./state.js` as needed (for reading state during rendering).
   - Functions to export (examine existing code for exact implementations):

   **Pure helpers (no state dependency):**
   - `showStatus(elementId, html)` — sets innerHTML of the status element
   - `clearStatus(elementId)` — clears innerHTML of the status element
   - `statusIcon(type)` — returns SVG icon HTML string for 'success', 'error', 'info'
   - `categoryIcon(category)` — returns emoji for category
   - `fmtSize(bytes)` — formats byte count to human-readable string
   - `timeAgo(ts)` — formats timestamp to relative time string
   - `esc(str)` — HTML-escapes a string
   - `buildErrorHtml(data)` — builds error HTML with optional details toggle (from Phase 2)

   **Rendering functions (read state):**
   - `showPage(page)` — switches visible page, updates nav active states
   - `renderSingleFileInfo(meta)` — renders file info card for single mode
   - `renderSingleFormats(fmts)` — renders format chip grid. IMPORTANT: Currently uses `btn.onclick = () => selectSingleFormat(f, btn)`. Change this to `btn.addEventListener('click', () => selectSingleFormat(f, btn))`. The `selectSingleFormat` function will be imported from `./single.js`. Handle the circular dependency concern: ui.js importing from single.js. If this creates a circular dep (single.js also imports from ui.js), use late binding: accept selectSingleFormat as a parameter or use a setter pattern.
   - `renderBulkList()` — renders the full bulk file list by calling renderBulkRow for each item
   - `renderBulkRow(item, index)` — renders a single bulk row. CRITICAL: Remove the `onchange="setBulkFormat(${i}, this.value)"` from the innerHTML template string. The format select should have NO inline handler — event delegation in main.js will handle it.
   - `updateBulkRowStatus(index, status, note)` — updates a bulk row's status display
   - `renderHistory()` — renders the history list from state
   - `initBulkListDelegation()` — attaches a single 'change' event listener on `#bulk-file-list` that delegates to setBulkFormat (imported from state.js) when a SELECT element changes. Uses `e.target.closest('[id^="bulk-row-"]')` to find row index.

   **Circular dependency resolution:**
   For `renderSingleFormats` needing `selectSingleFormat` from single.js while single.js imports from ui.js: Use a registration pattern. Export a `registerSelectSingleFormat(fn)` setter in ui.js. single.js calls `registerSelectSingleFormat(selectSingleFormat)` at import time. Then renderSingleFormats uses the registered function. This avoids circular imports.
