<task type="auto">
  <name>Task 1: Create single.js and bulk.js feature modules</name>
  <files>frontend/js/single.js, frontend/js/bulk.js</files>
  <action>
1. Create `frontend/js/single.js`.
   - Import needed functions from `./state.js`, `./api.js`, `./ui.js`.
   - Extract and export these functions from the existing script block:

   **File handling:**
   - `handleFiles(files)` — routes File objects to uploadSingle (if 1 file) or uploadBulk (if multiple). This is the central file router called by drag-and-drop and file input handlers.
   - `onFilePick(event)` — handles file input change event, calls handleFiles with event.target.files
   - `onDragOver(event)` — prevents default, adds drag-over class to drop zone
   - `onDrop(event)` — prevents default, removes drag-over class, calls handleFiles with event.dataTransfer.files

   **Single-file conversion flow:**
   - `uploadSingle(file)` — calls apiUpload, stores result in state via setSingleFile, calls renderSingleFileInfo and renderSingleFormats, switches to single-file view
   - `selectSingleFormat(format, btn)` — updates singleFormat in state, updates active chip styling. Register this with ui.js via `registerSelectSingleFormat(selectSingleFormat)` so renderSingleFormats can use it without circular imports.
   - `convertSingle()` — reads state (singleFileId, singleFormat, currentQuality), calls apiConvert, handles response (shows download link or error), adds to history
   - `resetAll()` — resets single and bulk state, clears UI, shows upload page

   - Each function should match the existing behavior EXACTLY. Read the current index.html script block carefully for every DOM manipulation, status message, error handling detail, and state update.
   - Use imports from state.js for state reads/writes, api.js for fetch calls, ui.js for DOM rendering.

2. Create `frontend/js/bulk.js`.
   - Import needed functions from `./state.js`, `./api.js`, `./ui.js`.
   - Extract and export these functions:

   **Bulk conversion flow:**
   - `uploadBulk(files)` — calls apiBulkUpload for each file (or single bulk endpoint), stores results in state via addBulkFile, renders bulk list, switches to bulk view
   - `applyGlobalFormat()` — reads the global format selector value, applies it to all compatible bulk files in state, re-renders bulk list
   - `convertBulk()` — reads bulkFiles from state, calls apiBulkConvert, updates each row status, adds successes to history

   - Match existing behavior EXACTLY. Pay close attention to:
     - How uploadBulk handles the response (it calls /api/upload per file, not /api/bulk-upload)
     - How convertBulk constructs the request body
     - How per-file success/error is displayed via updateBulkRowStatus
     - How the global format selector interacts with individual row selectors

CRITICAL NOTES:
- All import paths MUST include `.js` extension (e.g., `import { foo } from './state.js'`).
- Do NOT put any functions on `window` — all cross-module communication goes through ES module imports.
- single.js must call `registerSelectSingleFormat(selectSingleFormat)` (imported from ui.js) at module load time so that ui.js can reference it without circular imports.
  </action>
